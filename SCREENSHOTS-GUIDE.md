# 📸 Screenshots Guide

This guide will help you capture and organize screenshots for your README.md

## 📁 Directory Structure

Create a `screenshots` folder in the root directory and add the following images:

```
screenshots/
├── home-feed.png          # Main feed with posts
├── profile.png            # User profile page
├── activity.png           # Activity dashboard
├── dark-mode.png          # Dark mode showcase
├── login.png              # Login page
├── signup.png             # Signup page
├── post-creation.png      # Create post modal
├── post-detail.png        # Single post view
├── comments.png           # Comments section
├── settings.png           # Settings page
├── mobile-view.png        # Mobile responsive view
└── features-grid.png      # Feature showcase
```

## 📷 What to Capture

### 1. Home Feed (`home-feed.png`)
**Capture:**
- Main feed with multiple posts
- Left sidebar with navigation
- Right sidebar with suggested users
- Create post section
- Stories section (if visible)

**Tips:**
- Make sure you're logged in
- Have at least 3-4 posts visible
- Show variety: text post, image post, poll
- Use a clean, professional account

### 2. Profile Page (`profile.png`)
**Capture:**
- Profile header with cover image
- Profile picture
- User bio and information
- Posts/Saved tabs
- Follow/Edit button
- User's posts grid

**Tips:**
- Use a complete profile with all fields filled
- Show follower/following counts
- Include 2-3 posts in the feed

### 3. Activity Dashboard (`activity.png`)
**Capture:**
- Activity stats at the top
- Activity list with different types:
  - New follower
  - Post like
  - Comment
  - Reply
- Activity filters (All, Likes, Comments, Follows)

**Tips:**
- Generate some activity before capturing
- Show variety of activity types
- Include timestamps

### 4. Dark Mode (`dark-mode.png`)
**Capture:**
- Same view as home feed but in dark mode
- Show the theme toggle
- Highlight the color scheme

**Tips:**
- Toggle to dark mode in settings
- Capture the same view as home-feed for comparison
- Show smooth color transitions

### 5. Login Page (`login.png`)
**Capture:**
- Login form
- Email and password fields
- "Remember me" checkbox
- Social login buttons (if any)
- "Forgot password" link
- "Sign up" link

**Tips:**
- Use a clean, empty form
- Show validation states if possible

### 6. Signup Page (`signup.png`)
**Capture:**
- Registration form
- All input fields
- Password strength indicator
- Terms and conditions checkbox
- "Already have an account" link

### 7. Post Creation (`post-creation.png`)
**Capture:**
- Create post modal
- Text input area
- Post type options (Text, Image, Video, Poll, Event)
- Image/video upload area
- Post button

**Tips:**
- Show the modal open
- Display all post type options
- Include emoji picker if visible

### 8. Post Detail (`post-detail.png`)
**Capture:**
- Single post view
- Post content (image/video/poll)
- Like, comment, share buttons
- Comments section
- Related posts (if any)

**Tips:**
- Choose a post with engagement
- Show comments and replies
- Include like count

### 9. Comments Section (`comments.png`)
**Capture:**
- Multiple comments
- Nested replies
- Comment likes
- "Reply" and "Delete" options
- Comment input field

**Tips:**
- Show at least 3-4 comments
- Include at least one reply
- Show comment timestamps

### 10. Settings Page (`settings.png`)
**Capture:**
- Settings sidebar navigation
- Active settings panel (Account/Security/Privacy)
- Form fields
- Save button

**Tips:**
- Show the full settings layout
- Highlight important options
- Use Account or Appearance settings

### 11. Mobile View (`mobile-view.png`)
**Capture:**
- Mobile responsive layout
- Hamburger menu (if any)
- Bottom navigation (if any)
- Stacked layout
- Touch-friendly buttons

**Tips:**
- Use browser DevTools mobile view
- Choose iPhone 12 Pro or similar
- Show portrait orientation
- Capture home feed or profile

### 12. Features Grid (`features-grid.png`)
**Create a collage showing:**
- Poll feature
- Event feature
- Video post
- Image post
- Save post feature
- Share feature

**Tips:**
- Use a photo editor to create a grid
- 2x3 or 3x2 layout
- Add labels for each feature
- Keep consistent sizing

## 🎨 Screenshot Best Practices

### Quality
- **Resolution:** Minimum 1920x1080 for desktop views
- **Format:** PNG for better quality
- **File size:** Optimize to < 500KB per image
- **Clarity:** Ensure text is readable

### Content
- **Use realistic data:** No "test" or "lorem ipsum"
- **Professional usernames:** Avoid inappropriate names
- **Clean UI:** No browser extensions visible
- **Consistent theme:** Use same theme across screenshots
- **No personal info:** Blur or remove real emails/phone numbers

### Browser
- **Use Chrome or Firefox:** Clean UI
- **Hide bookmarks bar:** Cleaner look
- **Full screen mode:** F11 for immersive view
- **Zoom level:** 100% (Ctrl+0)

### Tools for Capturing

**Windows:**
- Snipping Tool (Win + Shift + S)
- ShareX (free, advanced)
- Greenshot (free)

**Mac:**
- Cmd + Shift + 4 (area selection)
- Cmd + Shift + 3 (full screen)
- CleanShot X (paid, professional)

**Browser Extensions:**
- Awesome Screenshot
- Nimbus Screenshot
- Full Page Screen Capture

### Editing Tools

**Free:**
- GIMP
- Paint.NET
- Photopea (online)

**Paid:**
- Adobe Photoshop
- Figma (free for individuals)
- Canva (free tier available)

## 🎬 Optional: Create a Demo GIF

For even better presentation, create animated GIFs showing:
- Login flow
- Creating a post
- Liking and commenting
- Following a user
- Switching themes

**Tools:**
- ScreenToGif (Windows, free)
- LICEcap (Mac/Windows, free)
- Kap (Mac, free)
- Gifox (Mac, paid)

**Tips for GIFs:**
- Keep under 10 seconds
- Optimize file size (< 5MB)
- Show smooth interactions
- Use 15-30 FPS
- Crop to relevant area only
